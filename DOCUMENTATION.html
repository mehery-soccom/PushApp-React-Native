The Mehery Event Sender SDK provides a comprehensive suite of tools to engage your users through smart notifications and interactive polls. Optimized for performance and high engagement.

📋 Prerequisites

Critical Checklist

Your app will not receive notifications if these foundations are missing:

  • Firebase Configuration: Correct placement of google-services.json and GoogleService-Info.plist.
  • Gradle Integration: Google Services plugin enabled in both build files.
  • Dependencies: Firebase Messaging and Async Storage must be installed.
  • Permissions: POST_NOTIFICATIONS and INTERNET permissions granted.

🛠 Installation

# Install core library
npm install react-native-mehery-event-sender

# Install peer dependencies
npm install @react-native-firebase/app @react-native-firebase/messaging
npm install @react-native-async-storage/async-storage react-native-push-notification

For iOS projects:

cd ios && pod install

⚙️ Configuration

Android Setup

Update android/build.gradle:

buildscript {
  dependencies {
    classpath 'com.google.gms:google-services:4.3.15'
  }
}

iOS Capabilities

Enable Push Notifications and Background Modes in Xcode, then update Info.plist:

<key>UIBackgroundModes</key>
<array>
  <string>remote-notification</string>
</array>

🔍 Event Tracking

The SDK allows capturing rich behavioral data:

Method Purpose
OnUserLogin(id) Links the device to a specific user identity.
updateUserProfile(data) Syncs user metadata for segmentation.
OnPageOpen(name) Tracks screen navigation and analytics.
sendCustomEvent(tag, meta) Sends business-specific custom interaction data.

Android Notification Keys

  • Carousel: imageUrls, carousel_images
